## 四、 保姆级API接入实战：以Python为例

现在，我们进入核心环节。我将以最流行的Python语言为例，演示如何连接测试网API，并完成一个完整的“查询-下单-风控”流程。

**环境准备**：确保你的Python环境已安装 `requests` 和 `python-dotenv` 库。建议使用虚拟环境。

```bash

```

1.  **安全存储密钥**：
    在项目根目录创建 `.env` 文件（确保已加入 `.gitignore`），填入你的测试网密钥：
    ```
    TESTNET_API_KEY=your_api_key_here
    TESTNET_SECRET_KEY=your_secret_key_here
    ```

2.  **编写基础请求函数**：
    创建一个 `binance_testnet.py` 文件，编写处理签名和发送请求的辅助函数。币安API要求对部分请求参数进行HMAC SHA256签名。

    ```python
    import hashlib
    import hmac
    import time
    import requests
    from urllib.parse import urlencode
    import os
    from dotenv import load_dotenv

    load_dotenv()  # 加载环境变量

    BASE_URL = 'https://testnet.binancefuture.com'
    API_KEY = os.getenv('TESTNET_API_KEY')
    SECRET_KEY = os.getenv('TESTNET_SECRET_KEY')

    def get_signed_params(params):
        """生成带签名的参数字符串"""
        query_string = urlencode(params)
        signature = hmac.new(SECRET_KEY.encode('utf-8'), query_string.encode('utf-8'), hashlib.sha256).hexdigest()
        params['signature'] = signature
        return params

    def send_signed_request(method, endpoint, params={}):
        """发送需要签名的请求"""
        url = BASE_URL + endpoint
        params['timestamp'] = int(time.time() * 1000)
        params = get_signed_params(params)
        headers = {'X-MBX-APIKEY': API_KEY}
        response = requests.request(method, url, headers=headers, params=params)
        return response.json()

    def send_public_request(endpoint, params={}):
        """发送公开请求（无需签名）"""
        url = BASE_URL + endpoint
        response = requests.get(url, params=params)
        return response.json()
    ```

3.  **实战操作一：查询账户信息与行情**：
    在开始交易前，先确认连接成功并获取市场数据。

    ```python
    # 1. 测试连接
    print(send_public_request('/fapi/v1/ping'))  # 应返回 {}
    
    # 2. 获取BTCUSDT永续合约的深度信息
    depth = send_public_request('/fapi/v1/depth', {'symbol': 'BTCUSDT', 'limit': 5})
    print("BTCUSDT买卖盘前五档：", depth)
    
    # 3. 查询测试账户资产（需要签名）
    account_info = send_signed_request('GET', '/fapi/v2/account')
    print("账户资产概况：", account_info['assets'])
    ```

4.  **实战操作二：下达你的第一笔测试订单**：
    我们下一个限价单来测试交易接口。

    ```python
    # 设置订单参数
    order_params = {
        'symbol': 'BTCUSDT',
        'side': 'BUY',  # 买入
        'type': 'LIMIT',
        'timeInForce': 'GTC',  # 成交为止，订单一直有效
        'quantity': 0.001,  # 数量，根据测试资产调整
        'price': 30000  # 限价，请设置为当前市价下方一个安全位置，以便测试
    }
    
    # 下单
    order_result = send_signed_request('POST', '/fapi/v1/order', order_params)
    print("订单结果：", order_result)
    
    # 查询订单状态
    if 'orderId' in order_result:
        check_params = {'symbol': 'BTCUSDT', 'orderId': order_result['orderId']}
        order_status = send_signed_request('GET', '/fapi/v1/order', check_params)
        print("订单状态：", order_status)
    ```
    **关键点**：这里的 `price` 需要你根据实时行情手动设定一个较低的值，以确保订单能进入订单簿但不会立即成交，方便你观察和后续撤单。

5.  **实战操作三：设置止损与撤单（风控演练）**：
    成熟的交易必须包含风控。我们为刚才的挂单设置一个止损单，然后撤销原订单。

    ```python

    stop_params = {
        'symbol': 'BTCUSDT',
        'side': 'SELL',
        'type': 'STOP_MARKET',  # 市价止损单
        'quantity': 0.001,
        'stopPrice': 29500,
        'closePosition': 'true'  # 平仓所有该合约头寸，更精确
    }
    # 注意：止损单需要在有持仓或挂单时设置，此处仅为示例逻辑。实际中，止损单常与开仓单同时提交或持仓后设置。
    
    # 2. 撤销之前挂的限价单

        cancel_params = {'symbol': 'BTCUSLT', 'orderId': order_result['orderId']}
        cancel_status = send_signed_request('DELETE', '/fapi/v1/order', cancel_params)
        print("撤单结果：", cancel_status)
    ```
