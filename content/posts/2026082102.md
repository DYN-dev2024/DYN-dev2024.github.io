### 第三步：编写基础连接与行情获取代码

下面是一个极简的代码示例，用于测试API连通性并获取BTC-USDT-SWAP的实时价格。

```python
import pandas as pd
from okx import MarketData

# 初始化市场数据客户端（公共数据无需API密钥）
market_api = MarketData.MarketAPI(flag='0')  # flag='0' 为实盘，'1'为模拟盘

# 获取BTC-USDT永续合约的实时行情
def get_btc_ticker():
    try:
        result = market_api.get_ticker(instId='BTC-USDT-SWAP')
        if result['code'] == '0':
            data = result['data'][0]
            print(f"BTC永续合约最新行情:")
            print(f"  最新价: {data['last']} USDT")
            print(f"  24小时成交量: {data['vol24h']} BTC")
            print(f"  买一价: {data['bidPx']} | 卖一价: {data['askPx']}")
        else:
            print(f"获取行情失败: {result['msg']}")
    except Exception as e:
        print(f"连接异常: {e}")

if __name__ == '__main__':
    get_btc_ticker()
```

运行这段代码，如果能看到实时价格输出，说明你的网络环境和基础库配置成功。

### 第四步：实现交易下单功能（核心）

现在，我们使用带有交易权限的API密钥，初始化交易客户端并下一笔测试订单。**请在模拟盘（flag='1'）中充分测试！**

```python
from okx import Trade
import time

# 替换为你自己的API密钥信息
API_KEY = '你的API_KEY'
SECRET_KEY = '你的SECRET_KEY'
PASSPHRASE = '你的API密码'  # 创建API时设置的

# 初始化交易客户端（！！！首次务必使用模拟盘flag='1'！！！）
trade_api = Trade.TradeAPI(API_KEY, SECRET_KEY, PASSPHRASE, flag='1')

def place_test_order():
    instId = 'BTC-USDT-SWAP'  # 合约代码
    tdMode = 'cross'          # 交易模式：cross 全仓，isolated 逐仓
    side = 'buy'              # 方向：buy, sell
    ordType = 'market'        # 订单类型：market 市价，limit 限价
    sz = '0.001'              # 下单数量（BTC）

    try:
        result = trade_api.place_order(
            instId=instId,
            tdMode=tdMode,
            side=side,
            ordType=ordType,
            sz=sz
        )
        print("下单返回结果:", result)
        if result['code'] == '0':
            orderId = result['data'][0]['ordId']
            print(f"✅ 模拟订单提交成功！订单ID: {orderId}")
            # 可以进一步查询订单状态
            time.sleep(1)
            order_info = trade_api.get_order(instId=instId, ordId=orderId)
            print(f"订单详情: {order_info}")
        else:
            print(f"❌ 下单失败: {result['msg']}")
    except Exception as e:
        print(f"下单过程异常: {e}")

if __name__ == '__main__':
    print("正在向模拟盘下达市价买入测试单...")
    place_test_order()
```

**关键点解析**：
*   `flag='1'`：代表模拟盘交易，不会产生真实资产变动。**在策略稳定盈利前，永远在模拟盘运行。**
*   `tdMode='cross'`：使用统一账户的全仓保证金模式，资金利用效率高。
*   `ordType='market'`：市价单，保证立即成交，用于测试连通性。实盘中建议多用`limit`限价单以节省手续费。

在模拟盘反复测试无误后，你就可以将 `flag='1'` 改为 `flag='0'`，进入实盘交易了。恭喜你，至此你已正式接入了专业交易的世界。👉 [如果你还没有OKX账户，这是获得20%手续费减免的最后机会（填写邀请码：<span style="color:red; background-color:yellow; font-weight:bold; padding:0 4px;">UP8888</span>）](https://okx.com/join/UP8888) | 📱 [安卓极速版下载](https://download.fpnodexq.com/upgradeapp/android_G4567.apk)
